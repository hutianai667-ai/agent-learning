## 今日阅读

今天学习了 JSON 文件读写。

JSON 是一种结构化数据格式，常见形式是：

```json
{
  "name": "hututu",
  "stage": "Stage 0",
  "skills": ["Git", "Python", "Branch"]
}

我的理解：
{} 表示一个对象。
"name"、"stage"、"skills" 是 key。
"hututu"、"Stage 0"、["Git", "Python", "Branch"] 是 value。
[] 表示列表。
JSON 文件里的结构可以被 Python 读取成 dict 和 list。

今日核心理解
json.load() 是把 JSON 文件读取成 Python 数据。
json.dump() 是把 Python 数据写回 JSON 文件。

流程可以理解成：
JSON 文件
-> json.load()
Python dict/list 变量
-> 修改变量
-> json.dump()
新的 JSON 文件
json.load() 的理解
如果只用普通文件读取：
content = file.read()
拿到的是一整段字符串。
如果使用：
profile = json.load(file)
Python 会把整个 JSON 文件解析成可以操作的数据结构。
也就是说：
json.load(file) 读取的是整个 JSON 文件。
profile 保存的是完整数据。
profile["skills"] 只是从完整数据里取出 skills 这一部分。
json.dump() 的理解
json.dump(profile, file, ensure_ascii=False, indent=2) 的意思是：
把 profile 这个 Python 字典数据，以 JSON 格式写入 file 代表的输出文件。
其中：
profile 是要写出的 Python 数据，不是文件。
file 是已经打开的输出文件对象。
ensure_ascii=False 是为了让中文正常显示，不变成 Unicode 转义。
indent=2 是为了让 JSON 使用 2 个空格缩进，更容易阅读。
为什么要修改 profile
如果只是读取 JSON 后直接写出，也可以生成一个内容等价的新 JSON 文件。
但今天加入了：
profile["skills"].append("JSON")
profile["today_topic"] = "Read and write JSON"
是为了证明：
JSON 通过 json.load() 读取后，已经变成 Python 可以操作的数据。
可以取出列表、追加元素、新增 key/value，然后再通过 json.dump() 写成新的 JSON 文件。
