## 今日阅读

今天学习了 `git clone`：

- `git clone` 可以从 GitHub 下载一个已有仓库到本地。
- clone 下来的目录已经是 Git 仓库，不需要再 `git init`。
- clone 后会自动配置 `origin`。
- `git remote -v` 可以查看 `origin` 指向的 GitHub 仓库地址。

`clone-practice-2026-06-17.md`

然后完成了：

1. `git clone`
2. `git remote -v`
3. 修改文件
4. `git add`
5. `git commit`
6. `git push`
7. 回到主仓库执行 `git pull`

## 今日理解

`git init` 是把本地普通文件夹变成 Git 仓库。

`git clone` 是从 GitHub 复制一个已有仓库到本地，并自动带上 Git 历史和远程地址 `origin`。
