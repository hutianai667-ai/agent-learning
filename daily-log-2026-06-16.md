## PR 合并与分支清理

今天完成了第一次 Pull Request。

流程：

1. 在本地创建分支 `branch-practice-2026-06-16`
2. 在分支上新增学习记录
3. commit 并 push 到 GitHub 远程分支
4. 在 GitHub 创建 Pull Request
5. 确认 `base: main`，`compare: branch-practice-2026-06-16`
6. merge PR 到 `main`
7. 本地切回 `main`
8. 执行 `git pull` 同步远程 main
9. 删除本地分支
10. 删除远程分支

今日理解：

- `origin` 是远程 GitHub 仓库的默认名称。
- `git push -u origin 分支名` 会把本地分支推送到 GitHub，并建立本地分支和远程分支的对应关系。
- `git push` 不是固定推到 `main`，而是推到当前分支对应的远程分支。
- PR 是把分支里的修改申请合并到 `main`。
- 合并后需要在本地 `git pull`，让本地 `main` 同步 GitHub 上最新的 `main`。
- 分支合并后可以删除，保持仓库干净。

今日踩坑：

- `git switch man` 是拼写错误，正确是 `git switch main`。

- 删除分支前要先切回 `main`。
