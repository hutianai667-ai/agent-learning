## 今天阅读

今天学习了 system prompt 和普通任务 prompt 的区别。

System prompt 用来设定长期稳定规则，比如角色、安全边界、输出风格、改文件规则和复盘方式。

普通任务 prompt 用来描述一次性具体任务，比如解释某个概念、创建某个文件、改写某段内容。

## 今天实操

今天完成了 system prompt 分类练习：

- 长期角色设定适合 system prompt
- 安全边界适合 system prompt
- 输出语言和协作规则适合 system prompt
- 当天具体任务适合普通任务 prompt

## 今天理解

安全边界适合写进 system prompt，因为它不是一次性要求，而是长期规则。

改文件前先 `git status` 是安全检查，可以确认是否有未暂存、未提交或误修改的文件。

`git push` 会把本地 commit 上传到远程仓库，因此不应该让 Agent 自动执行，必须由用户确认。
