## 今天阅读

今天学习了 A1 的目标：让 CLI agent 开始做真实、低风险、可验证的小任务。

低风险任务的特点是：

- 只读文件
- 不修改配置
- 不删除文件
- 不上传 GitHub
- 不读取 `.env`、API key、密码或隐私文件
- 输出可以人工检查

## Agent 使用记录

- 任务名称：总结学习仓库结构和已有笔记
- 任务风险：低风险，只读任务
- agent 是否先说明计划：有简短说明，但不够具体；没有先列出具体准备读取的目录和文件让我确认
- agent 读取了哪些文件：主要读取 `.md` 学习笔记标题和文件列表
- agent 有没有修改文件：没有
- 输出哪里可用：仓库概览、Stage 2 文件检查、当前进度判断
- 输出哪里不可用：后续学习建议偏泛，不够具体，实操性不强
- 我的人工检查：确认任务没有修改文件，并判断输出质量

## 今天理解

CLI agent 和网页聊天最大的区别是：CLI agent 可以读取和操作本机文件，所以必须设置安全边界。

A1 阶段不只是让 agent 完成任务，更重要的是观察：

- 它有没有先说明计划
- 它读取了什么
- 它是否越界
- 它是否修改文件
- 输出是否具体、可检查、可执行
