## 今日关键理解补充

`enumerate(items, start=1)` 的调用参数和它每轮产出的结果不是一回事。

- 调用时：`items` 表示要遍历的列表，`start=1` 表示编号从 1 开始。
- 产出时：每一轮得到的是 `(编号, 元素)`。
- 所以 `for index, item in enumerate(items, start=1)` 里，`index` 接编号，`item` 接元素。

函数传参不是靠名字必须一样，而是靠调用时的位置。

例如：

```python
def build_line(index, item):
    return f"{index}. I learned {item}"

for number, text in enumerate(items, start=1):
    line = build_line(number, text)

### PowerShell / Git

`python week-01\scripts\hello.py` 的意思是：用 Python 运行这个路径下的 `hello.py` 脚本。

`git add` 不是上传，而是把文件加入暂存区，准备下一次 `commit`。

`git commit -m "说明"` 是把暂存区里的修改保存成一次本地记录，并附带说明。

`git push` 才是把本地 commit 推送到 GitHub 远程仓库。

判断一次练习是否保存并上传成功，要看两点：

1. `git push` 没有报错。
2. `git status` 最后显示 `nothing to commit, working tree clean`。

### Python 字符串和函数

f-string 的准确理解：

`f"..."` 表示格式化字符串，`{}` 里的变量或表达式会被替换成真实值。

例如：

```python
f"{index}. {item}"
会把 index 和 item 替换成当前变量值。
def build_line(index, item): 的意思是：定义一个叫 build_line 的函数，它接收两个参数 index 和 item。
函数传参主要靠位置，不要求外部变量名和函数内部参数名一样。
例如：
def build(number, text):
    return f"{number}. {text}"

for index, item in enumerate(items, start=1):
    line = build(index, item)
这里 index 的值传给 number，item 的值传给 text。
list / for
items = [] 的意思是：创建一个叫 items 的空列表。
items.append("Git") 的意思是：把 "Git" 这个元素添加到 items 列表最后。
enumerate(items, start=1) 会在遍历 items 时，同时提供两个值：
编号
当前元素
所以：
for index, item in enumerate(items, start=1):
表示：每一轮循环里，index 接编号，item 接当前元素。
文件读写
with open(path, "r", encoding="utf-8") as file: 的意思是：
打开 path 指向的文件，使用指定模式和编码；把打开后的文件对象临时命名为 file；with 代码块结束后自动关闭文件。
file.write(summary) 的意思是：把 summary 变量里的内容写入 file 代表的已打开文件。
写文件前必须先 open()，因为 Python 需要知道：

使用什么模式
使用什么编码
当前打开的文件对象叫什么
今日易错点
index + item 会报错，因为 index 是数字，item 是字符串，Python 不能直接用 + 拼接数字和字符串。
正确写法：
f"{index}. {item}"
或者：
str(index) + ". " + item
summary = summary + line + "\n" 的意思是：把旧的 summary、当前这一行 line、换行符拼在一起，再保存回 summary，实现逐行累积文本。
