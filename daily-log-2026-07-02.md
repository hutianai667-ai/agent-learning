## 今天阅读

今天学习了 CoT prompt。

CoT 是 Chain of Thought，可以理解为让模型在复杂任务中先检查关键条件，再给出结论。

今天重点理解了：CoT 不等于要求模型输出完整思考过程。更实用的方式是让模型检查关键条件，但最终只输出结论和简短理由。

## 今天实操

今天完成了一个 CoT 分类 prompt 练习。

待分类输入是：

```text
今天我学习了 .gitignore 规则和 .env.example 模板。


这个输入存在分类冲突：
.gitignore 偏 Git
.env.example 偏 Security
所以 prompt 里加入了检查步骤：
是否涉及安全内容
是否涉及 Git 内容
如果 Git 和安全冲突，优先 Security
不输出完整检查过程，只输出最终类别和简短理由
预期输出是：
类别：Security
简短理由：同时涉及 Git 忽略规则和密钥安全模板，按安全优先规则分类。
今天理解
CoT 的目标是提高复杂任务的判断质量，不是让模型输出长篇推理。
简单任务不需要 CoT，比如规则明确的分类、翻译、字段提取。
适合用 CoT 的场景包括：
分类规则有冲突
需要比较几个选项
需要排查错误原因
直接回答容易漏掉关键条件
今天纠正
我理解到 .gitignore 和 .env.example 的冲突，不是因为它们都属于安全，而是因为：
.gitignore -> Git
.env.example -> Security
当两类同时出现时，需要使用优先级规则，最终选择 Security。
安全边界
今天没有读取、输出、修改 .env、API key、密码文件或隐私文件。
