## 今天阅读

今天学习了 API key / token 不能上传 GitHub。
真实密钥应该放在本地 `.env` 文件中，但 `.env` 必须被 `.gitignore` 忽略。

## 今天实操

- 创建 `.gitignore`
- 在 `.gitignore` 中配置忽略规则：
  - `.env`
  - `.env.*`
  - `!.env.example`
- 创建 `.env.example` 作为安全模板
- 使用 `git check-ignore -v --no-index .env` 检查 `.env` 是否会被 Git 忽略

## 今天理解

`.gitignore` 是 Git 约定好的忽略规则文件名，必须叫这个名字才会被 Git 自动读取。

`git check-ignore` 是检查忽略规则是否生效。

`-v` 用来显示是哪一条规则生效。

`--no-index` 表示即使 `.env` 文件现在不存在，也可以检查规则是否会匹配它。

`.env` 不能上传，`.env.example` 可以上传，因为它只放占位符，不放真实 key。
