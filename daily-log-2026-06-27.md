## 今天阅读

今天学习了 temperature 的作用。

Temperature 是控制模型输出随机程度的参数。低 temperature 更稳定、更保守，适合代码、排错、总结和 CLI Agent 执行任务。高 temperature 更发散、更有创意，适合头脑风暴、起名字、文案改写。

## 今天实操

今天完成了 temperature 判断练习：

- 修改 Python bug：低 temperature
- 想 20 个 AI 学习计划名字：高 temperature
- 总结错误日志：低 temperature
- 写 5 种广告文案风格：高 temperature
- 修改 GitHub 仓库文件：低 temperature

今天还为 `context window` 写了两个 prompt：

- 稳定版 prompt：适合低 temperature，目标明确、限制清楚、格式固定
- 创意版 prompt：适合高 temperature，需要类比、生动表达和多角度解释

## 今天理解

真实执行类任务更适合低 temperature，因为需要稳定、准确和可控。

创意发散类任务更适合高 temperature，因为需要多样性和不同角度。

Temperature 不是正确性的保证。输出质量还取决于 prompt 是否清楚、上下文是否相关、模型能力和是否有验证步骤。

## 加练：Stage 1 第 4 节 Prompt 结构化改写

今天继续完成了 Stage 1 第 4 节，练习把弱 prompt 改写成结构化 prompt。

今天理解到，一个好的 prompt 不只是提问题，而是给模型一份任务说明。它应该包含角色、任务、背景、读取范围、安全限制、输出格式和验收标准。

CLI Agent 任务尤其需要明确读取范围和安全限制，避免读取无关文件、浪费 token，或者误读 `.env`、API key、密码文件。

今天还练习了省 token prompt。省 token 不是把内容越删越短越好，而是保留核心限制，删除多余解释。
