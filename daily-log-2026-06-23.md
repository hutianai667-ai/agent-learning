我的理解：

- GitHub API 返回的数据是 JSON。
- Python 可以请求 API，然后用 `json.load()` 把返回的 JSON 转成 Python 数据。

今日核心理解
urllib.request.urlopen(url) 会向网络地址发送请求，并打开返回的响应。
response 是 API 返回的响应对象，可以理解为 GitHub 返回的数据流。
json.load(response) 和昨天的 json.load(file) 类似：
json.load(file) 读取本地 JSON 文件
json.load(response) 读取网络 API 返回的 JSON
result 是一个 Python 字典，用来保存我关心的字段。
json.dump() 可以理解为把 Python 数据写出去，写成 JSON 文件。
