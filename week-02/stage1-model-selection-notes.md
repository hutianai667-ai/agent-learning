# Stage 1 - Model Selection Notes

## 1. 核心原则

不是所有任务都要用最强模型，也不是所有任务都需要 AI。

选择模型时要看：

- 任务复杂度
- 风险高低
- 是否需要推理
- 是否有明确规则
- 是否涉及隐私或敏感信息
- 输出价值是否足够高

## 2. 什么时候用强模型

适合使用强模型的任务：

- 复杂推理
- 多文件代码理解
- 难 bug 分析
- 技术架构设计
- 安全风险判断
- 高价值任务
- 复杂长文总结

注意：

长文总结不一定都用强模型，要看内容复杂度和价值。
短但高风险的任务，也可能需要强模型加人工确认。

## 3. 什么时候用便宜模型

适合使用便宜模型的任务：

- 简单总结
- 格式转换
- 改写语气
- 生成模板
- 分类
- 提取字段
- 简单问答
- 少量文字总结

## 4. 什么时候不用模型

适合不用模型的任务：

- 确定性命令
- 复制文件
- 重命名文件
- 简单搜索
- 固定规则替换
- 计算明确结果

这些任务用脚本、命令或普通工具通常更稳定。

## 5. 什么时候必须人工确认

必须人工确认的任务：

- 删除文件
- 修改数据库
- 发布生产代码
- 涉及账单
- 涉及隐私
- 涉及 API key、密码、`.env`
- 大规模自动修改
- 安全审查结果

这类任务不能完全交给模型自动决定。

## 6. 任务分类练习

| 任务 | 选择 | 原因 |
|---|---|---|
| 总结一篇 500 字学习笔记 | 便宜模型 | 内容短，任务简单 |
| 分析跨 5 个文件的复杂 bug | 强模型 | 需要多文件理解和推理 |
| Markdown 表格转 CSV | 便宜模型 / 不用模型 | 规则明确，脚本也能完成 |
| 删除整个 outputs 文件夹 | 人工确认 | 涉及删除文件 |
| 给新产品想 30 个名字 | 便宜模型 | 创意任务但风险低 |
| 判断代码是否泄露 API key | 强模型 + 人工确认 | 涉及安全风险 |
| 批量替换文件名空格 | 不用模型 | 固定规则任务 |
| 发布到生产环境 | 人工确认 | 高风险操作 |

## 7. 我的模型选择规则

以后做 AI Agent 任务时：

- 复杂推理、技术架构、安全审查、高价值任务，用强模型。
- 固定格式输出、改写语气、生成模板、简单问答、短文本总结，用便宜模型。
- 确定性任务、复制文件、固定规则替换、简单搜索，不一定需要模型。
- 涉及隐私、API key、`.env`、账单、删除文件、生产发布，必须人工确认。
