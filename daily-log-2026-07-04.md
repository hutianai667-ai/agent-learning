## 今天阅读

今天学习了 prompt 迭代。

Prompt 迭代不是一次写出完美 prompt，而是：

```text
写一版 -> 测试 -> 发现问题 -> 改一个重点 -> 再测试 -> 记录变化
今天重点理解了：
每一版最好只改一个重点
使用同一条测试输入，方便前后对比
记录每版差异，比追求第一版完美更重要
今天实操
今天对同一个学习记录分类 prompt 做了 5 版迭代。
测试输入是：
今天我学习了 .gitignore 规则、.env.example 模板和 few-shot prompt。
这条输入同时涉及：
.gitignore -> Git
.env.example -> Security
few-shot prompt -> Prompt
所以它适合用来测试分类规则、冲突规则、few-shot 示例和检查步骤是否有效。
5 版迭代如下：
V1：基础版，只写角色、任务、类别、输入、输出格式、限制。
V2：加入分类规则，让分类依据更清楚。
V3：加入冲突规则，明确安全内容优先选择 Security。
V4：加入 few-shot 示例，让模型模仿普通分类、冲突分类和 Other 兜底。
V5：加入检查步骤，并输出类别和简短理由。
今天理解
Prompt 迭代的重点不是把 prompt 写得越来越长，而是每次针对一个问题做改进。
V1 容易不稳定，因为缺少分类规则和冲突规则。
V2 更清楚，但遇到多类别输入时仍可能冲突。
V3 通过 Security 优先规则解决冲突。
V4 用 few-shot 示例增强模型模仿能力。
V5 通过检查步骤和简短理由，让输出更可验证。
今天纠正
我理解到：分类规则并不能解决所有问题。
当一个输入同时命中多个类别时，还需要冲突规则或优先级规则。否则模型可能在多个类别之间摇摆。
