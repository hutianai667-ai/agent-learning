今天我理解了 `content`、`summary`、`output_path` 的区别：

- `content` 是从输入文件读取出来的原始内容。

- `output_path` 是输出文件路径，决定写到哪个文件。
- `summary` 这个变量名和 `day2-summary.txt` 文件名没有直接关系，只是名字刚好相似。
- `file.write(summary)` 是把 `summary` 变量里的内容写入已经打开的输出文件。


# Extra Log - 2026-06-11

## 今日预习内容

第 1 周第 3 天预习：Python 追加写入文件。

## 今日阅读

今天理解了 Python 文件打开模式：

- `"r"` 是读取。
- `"w"` 是写入，会覆盖原文件内容。
- `"a"` 是追加，会把新内容写到文件末尾，不会覆盖旧内容。

## 今日实操

今天创建了 `week-01/scripts/append_log.py`。

脚本流程：

1. 用 `input()` 接收一条学习笔记。
2. 用 `open(log_path, "a", encoding="utf-8")` 打开日志文件。
3. 用 `file.write(new_entry + "\n")` 把笔记追加到文件最后。
4. 运行两次后，`practice-log.txt` 里保留了两条记录。

## 今日理解

`"w"` 适合生成新结果，`"a"` 适合不断追加日志。

`new_entry + "\n"` 里的 `\n` 是换行，否则多次写入会挤在同一行。
